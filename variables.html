Variables

Variables are one of the most important concepts within programming. Variables allow for the programmer to gather user input, modify data, and even store data! In this lesson you will learn to create a few types of variables as well as how to use them!

Integers

Intregers are simply whole numbers! Intreger examples being 1, 10, 4, etc. Intregers within c++ are also unsigned, meaning they may be positive or negative. The unsigned keyword may negate this allowing for only positive numbers to be held within an intreger variable. Lets look at an example of the decleration of an intreger.


#include
using namespace std;
int main(){
    int variableName;
    int variableNameTwo = 1;
    cout << variableName << endl;
}

Characters

Characters, also known as char, allows us to store single characters. In a later lesson you will learn to combine these characters in order to form a c-string! The char data type can store any single character ('A', 'b', '1', '%', etc).

char myChar;
myChar = 'A';
char anotherChar = 'B';

Character literals may also be represented using their ASCII values

char myChar = 65; // Assigning the ASCII value 65, which corresponds to 'A'

Boolean

In C++, a boolean represents true or false. Booleans are mainly used for controlling the flow of programs through conditional statements (covered in future lesson). A boolean will always represent False : {0} or True : {1}. The keyword true or false may be used for assignment.

bool isSunny = true;
isSunny = 0;
The above statement demonstrates how booleans may be implemented within a program. Booleans may be combined in order to acomplish a more complex function! While it is not conventionally how booleans are used we can use boolean arithmetic to create gates! Below is an example of an or gate created with boolean arithmetic.

bool isSunny = false;
bool isCloudy = false;
bool shouldIGoOutside = isSunny + isCloudy;

Floats

Floating-point numbers are used to store real numbers, they may allow fractional parts. Floats are built into c++, allowing for the storage of decimal numbers. They store 32 bits of memory (4 bytes) providing around 7 digits of precision. Beware that floats may provide inexact data due to their precision being limited. This is due to the trade off in memory and precision. A computer has limited storage capacity, so it must approximate based on the given bits. numbers such a 3.14 will appear as 3.1400000003232 (not a real example) because the number is approximated. For this reason it may be better to utalize doubles for more precise calculation.

float myFloat;
myFloat = 3.14;
float anotherFloat = 2.71828;
